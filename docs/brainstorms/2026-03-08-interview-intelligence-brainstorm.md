# Interview Intelligence — Brainstorm Session

## What We're Building

An agentic interview intelligence tool — agent-first architecture, self-hosted web app, open source. Anyone can clone it, run it locally with their own data, and deploy it.

**Not** a generic dashboard with agents bolted on in Phase 4. The agents are the product. The UI is thin but real — enough to manage your pipeline, view briefings, and upload transcripts.

---

## Problem Space

### Problem Space B: Interview Conversion (v1 scope)

> "Prep is scattered across 5-6 files, I can't see if my recurring issues are improving, I have no feedback loop from interview to interview."

**Specific pain points (from real usage across 8+ interviews):**
- Pre-interview prep takes 10 minutes of hunting through Obsidian files
- Gap analysis exists but no way to see trends over time
- Post-debrief insights disappear into flat files — no connection to bigger picture
- Story deployment not tracked — can't see which stories are overused or untested
- No single view that says "here are your 3 active issues and your trajectory"

### Problem Space A: Pipeline Generation (v2)

> "I don't know which companies to target, I'm not doing consistent outreach, and things fall through the cracks."

- Research Agent, Outreach Agent, Pipeline Agent
- Deferred to v2 — Research Agent partially built already

---

## The Four Agentic Concepts We're Demonstrating

| Concept | Where It Shows Up |
|---|---|
| Agentic loop + tool calling | All agents — each uses tools (web search, file read, DB write) in a loop |
| Task decomposition + planning | Prep Agent — decomposes "prep for interview" into subtasks with a plan |
| Multi-agent orchestration | Coach Agent — coordinates Prep + Debrief agents, manages state |
| Memory + context management | Debrief Agent — reads gap history, updates priorities, persists state |

---

## Architecture Decisions Made

**Stack:**
- Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS
- Database: Supabase (Postgres)
- Hosting: Self-hosted (local) → Vercel (production)
- Agents: Python, Claude API (Anthropic SDK)

**Deployment model:** Self-hosted web app (Option B). Clone repo, run locally, bring your own data. Designed to deploy to Vercel later for full production.

**Open source:** MIT license, public GitHub from day one.

**Data portability:** Users bring their own resume, stories, interview history. No lock-in to personal-os-main data.

---

## Agent Architecture (v1)

| Agent | Pattern | What It Does |
|---|---|---|
| Research Agent | ReAct + tool calling | ✅ Already built — company research on demand |
| Prep Agent | Plan-and-Execute | Given "Interview at [Company] - [Round]", decomposes into: fetch questions, match stories, check gaps, research interviewer → outputs briefing |
| Debrief Agent | Memory + context management | Reads transcript, scores answers, reads gap history, updates priorities |
| Coach Agent | Multi-agent orchestration | Coordinates Prep + Debrief, checks pipeline proactively, surfaces alerts |

---

## UI Scope (v1 — thin but real)

- Interview pipeline board (companies, stages, upcoming dates)
- Single-screen interview briefing (generated by Prep Agent)
- Transcript upload + debrief display (processed by Debrief Agent)
- Improvement tracker with trends (updated after each debrief)
- Story bank view (deployment history, health status)
- Question bank view (filterable by company, stage, category)

**Not in v1:**
- Mock interview mode
- Networking CRM / outreach tracking
- Calendar integration
- Multi-user auth
- Mobile app
